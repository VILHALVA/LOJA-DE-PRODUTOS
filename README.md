## DESCRIﾃﾃグ:
- Este projeto consiste em uma aplicaﾃｧﾃ｣o web desenvolvida utilizando PHP 8, MySQL, Laravel 9, MaterializeCSS, Darryldecode Cart e Chart.js. A proposta ﾃｩ criar uma plataforma semelhante ao Amazon, onde os usuﾃ｡rios podem cadastrar produtos com informaﾃｧﾃｵes como nome, preﾃｧo, descriﾃｧﾃ｣o, categoria e foto. Alﾃｩm disso, eles podem adicionar esses produtos a um carrinho de compras e realizar operaﾃｧﾃｵes bﾃ｡sicas de CRUD (criaﾃｧﾃ｣o, leitura, atualizaﾃｧﾃ｣o e exclusﾃ｣o) sobre os produtos.

- O projeto ﾃｩ implementado seguindo as melhores prﾃ｡ticas de desenvolvimento web e utiliza tecnologias modernas para proporcionar uma experiﾃｪncia de usuﾃ｡rio eficiente e segura. Com uma interface amigﾃ｡vel e recursos robustos, a aplicaﾃｧﾃ｣o oferece uma plataforma completa para gerenciar uma loja de produtos online.

## RECURSOS:
1. **Gerenciamento de Produtos:** 
   - Os usuﾃ｡rios podem cadastrar novos produtos fornecendo detalhes como nome, preﾃｧo, descriﾃｧﾃ｣o, categoria e imagem. O formulﾃ｡rio de cadastro de produtos ﾃｩ exibido em um modal, proporcionando uma experiﾃｪncia de usuﾃ｡rio intuitiva.

   - No painel de produtos, os usuﾃ｡rios tﾃｪm a capacidade de visualizar, adicionar e excluir produtos. Isso oferece controle total sobre o catﾃ｡logo de produtos da loja.

2. **Listagem de Produtos na Home:** Na pﾃ｡gina inicial, os usuﾃ｡rios podem visualizar uma lista de produtos disponﾃｭveis. A paginaﾃｧﾃ｣o ﾃｩ implementada para facilitar a navegaﾃｧﾃ｣o entre os produtos.

3. **Detalhes do Produto:** Ao clicar em um produto, os usuﾃ｡rios podem visualizar detalhes como nome, preﾃｧo, descriﾃｧﾃ｣o e categoria. Eles tambﾃｩm tﾃｪm a opﾃｧﾃ｣o de adicionar o produto ao carrinho de compras.

4. **Carrinho de Compras:** Os usuﾃ｡rios podem adicionar produtos ao carrinho de compras e visualizar o total de produtos no carrinho. Eles podem atualizar a quantidade de produtos no carrinho, remover produtos e limpar completamente o carrinho.

5. **Autenticaﾃｧﾃ｣o de Usuﾃ｡rios:** A autenticaﾃｧﾃ｣o de usuﾃ｡rios ﾃｩ implementada para restringir o acesso a determinadas pﾃ｡ginas, como o painel de estatﾃｭsticas (Dashboard). Os usuﾃ｡rios devem estar logados para acessar essas pﾃ｡ginas.

6. **Cadastro de Usuﾃ｡rios:** Os usuﾃ｡rios tﾃｪm a opﾃｧﾃ｣o de se registrar na plataforma para acessar recursos exclusivos, como o painel de administraﾃｧﾃ｣o. O formulﾃ｡rio de registro permite que os usuﾃ｡rios forneﾃｧam informaﾃｧﾃｵes bﾃ｡sicas, como nome, sobrenome, e-mail e senha.

7. **Logout de Usuﾃ｡rios:** Os usuﾃ｡rios tﾃｪm a opﾃｧﾃ｣o de fazer logout da plataforma a qualquer momento. Ao clicar no botﾃ｣o de logout, eles serﾃ｣o desconectados de suas contas e redirecionados para a pﾃ｡gina de home. 

8. **Seguranﾃｧa:** A pﾃ｡gina de login possui validaﾃｧﾃ｣o de entrada e autenticaﾃｧﾃ｣o de dois fatores para garantir a seguranﾃｧa dos usuﾃ｡rios. Alﾃｩm disso, as rotas do painel de administraﾃｧﾃ｣o sﾃ｣o protegidas por middleware para verificar se o usuﾃ｡rio estﾃ｡ autenticado.

9. **Painel de Estatistiscas (Dashboard):** Esse painel fornece estatﾃｭsticas sobre o desempenho da loja, incluindo faturamento, nﾃｺmero de usuﾃ｡rios e nﾃｺmero de pedidos. Grﾃ｡ficos interativos sﾃ｣o usados para visualizar essas estatﾃｭsticas.

10. **Proteﾃｧﾃ｣o contra Acesso Nﾃ｣o Autorizado:** Tentativas de acessar o painel de estatﾃｭsticas sem autenticaﾃｧﾃ｣o resultam em redirecionamento para a pﾃ｡gina de login. Isso garante que apenas usuﾃ｡rios autorizados possam acessar ﾃ｡reas restritas do aplicativo.

## EXECUTANDO O PROJETO:
**1. Configuraﾃｧﾃ｣o do arquivo `.env`:**
   - Antes de executar o aplicativo, ﾃｩ necessﾃ｡rio configurar o arquivo de ambiente `.env` com as informaﾃｧﾃｵes do seu banco de dados MySQL.
   - Abra o arquivo `.env` localizado na raiz do diretﾃｳrio do projeto `CODIGO` e insira as informaﾃｧﾃｵes do seu banco de dados conforme abaixo:
   ```plaintext
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=cursolaravel
   DB_USERNAME=seu_usuario
   DB_PASSWORD=sua_senha
   ```
   Certifique-se de substituir `seu_usuario` e `sua_senha` pelos detalhes de autenticaﾃｧﾃ｣o do seu banco de dados MySQL.

**2. Criaﾃｧﾃ｣o do Banco de Dados MySQL:**
   - Apﾃｳs configurar o arquivo `.env`, ﾃｩ necessﾃ｡rio criar o banco de dados MySQL com o nome especificado na configuraﾃｧﾃ｣o.
   - Execute o seguinte comando SQL no seu cliente MySQL ou na linha de comando:
   ```sql
   CREATE DATABASE cursolaravel CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
   Isso garantirﾃ｡ que o banco de dados seja criado com a codificaﾃｧﾃ｣o `utf8mb4_unicode_ci`, adequada para suportar caracteres multibyte e emojis, se necessﾃ｡rio.

**3. Instalando Dependﾃｪncias:**
   - Navegue atﾃｩ o diretﾃｳrio do projeto clonado no seu terminal ou prompt de comando.
   - Execute o seguinte comando para instalar todas as dependﾃｪncias listadas no arquivo `composer.json`:
   ```bash
   composer install
   ```
   Isso instalarﾃ｡ todas as dependﾃｪncias necessﾃ｡rias para o funcionamento correto do projeto, garantindo que todas as bibliotecas e pacotes estejam presentes e atualizadas.

**4. Gerenciando as Tabelas:**
   - As migraﾃｧﾃｵes jﾃ｡ foram criadas utilizando o comando `php artisan make:migration create_table_name --create=table_name...`. Vocﾃｪ pode encontrﾃ｡-las no diretﾃｳrio `./CODIGO/database/migrations`. Lembre-se de que vocﾃｪ pode editar esses arquivos antes de executﾃ｡-los.
   - Para aplicar as migraﾃｧﾃｵes ao banco de dados (Apenas para criar as Tabelas), utilize o seguinte comando:
   ```bash
   php artisan migrate
   ``` 
   Este comando executarﾃ｡ todas as migraﾃｧﾃｵes que ainda nﾃ｣o foram aplicadas, garantindo que o esquema do banco de dados esteja atualizado de acordo com suas definiﾃｧﾃｵes.

   - Para executar vﾃ｡rios seeders de uma vez (Inserir os registros nas Tabelas), vocﾃｪ pode usar o comando `db:seed` sem argumentos:
   ```bash
   php artisan db:seed
   ```
   Isso executarﾃ｡ todos os seeders registrados no mﾃｩtodo `run()` do arquivo `DatabaseSeeder.php`, localizado no diretﾃｳrio `database/seeders`. Atﾃｩ o momento, foram criados seeders apenas para as tabelas: `Produtos`, `Users` e `Categorias`. Vocﾃｪ pode criar outros para `regras`.

**5. Importaﾃｧﾃ｣o das Tabelas:**
   - Para acessar o conteﾃｺdo mostrado nas imagens, ﾃｩ necessﾃ｡rio importar o arquivo `DATABASE/DATABASE.sql` para o seu banco de dados. Este arquivo inclui exclusivamente as tabelas com seus conteﾃｺdos correspondentes.

**6. Criando um link simbﾃｳlico:**
   - Execute o seguinte comando:
   ```bash
   php artisan storage:link
   ```

   - Isso permite que os arquivos armazenados na pasta storage sejam acessﾃｭveis publicamente atravﾃｩs da URL public/storage (As fotos sﾃ｣o salvas em `CODIGO/storage/app/public/produtos`). 
   
**7. Executando o Servidor Embutido:**
   Agora vocﾃｪ pode iniciar o servidor embutido do PHP com o seguinte comando:
   ```bash
   php artisan serve
   ```
   Isso iniciarﾃ｡ o servidor local e vocﾃｪ poderﾃ｡ acessar o aplicativo por meio do navegador da web.

8. **Acessando o Site:**
   * Abra seu navegador da web favorito e digite o seguinte endereﾃｧo na barra de endereﾃｧos:
   ```
   http://localhost:8000
   ```
   * Agora vocﾃｪ pode explorar o site e interagir com os diferentes recursos clicando nos botﾃｵes disponﾃｭveis!
